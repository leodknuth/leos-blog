# resources 目录说明

此目录由 Hugo 在构建过程中用于存放生成的资源（build artifacts / cache）。常见内容包括：

- `resources/_gen`：Hugo 资源管线（images、assets 等）产生的衍生文件，例如缩放/裁剪后的图片、指纹化或压缩后的 CSS/JS 等。
- 其他由短代码或模板在构建时生成的临时/中间文件。

注意事项：

- 这些文件是构建产物（可由 Hugo 自动重建），通常不需要提交到版本控制系统。建议将 `resources/_gen` 加入 `.gitignore`（仓库默认已包含该项）。
- 如果需要回收磁盘空间，可以安全删除 `resources/_gen`，Hugo 会在下一次构建时重新生成所需文件。
- 在 CI 或自动部署场景，通常不需要把这些生成缓存提交到仓库。相反，应在 CI 中安装 Hugo 并在构建步骤中生成静态 `public/` 输出，然后将 `public/` 部署到静态托管服务。

简短命令参考：

```bash
# 删除生成缓存（会彻底删除目录）
rm -rf resources/_gen

# 本地重建（会在需要时重新生成）
hugo -s . --minify
```
